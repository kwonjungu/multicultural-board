# 🍯 칭찬 스티커 시스템 — 설계 계획서

**한 줄 요약**: 선생님이 준 칭찬 스티커 = 꿀벌 한 마리. 모으면 꿀벌집이 차고, 개인은 알→애벌레→번데기→벌→여왕벌로 성장한다.

---

## 1. 콘셉트 & UX 목표

### 핵심 은유
- **스티커 1개 = 꿀벌 1마리 = 꿀벌집의 육각 셀 1칸**
- 모일수록 **꿀벌집이 채워진다** (시각적 누적감)
- 각 학생은 **나만의 성장 단계**를 거친다 (정체성 + 동기)

### 학생이 보는 것 (개인 LMS 뷰)
1. **내 꿀벌집** — 스티커 누적을 육각 셀로 시각화
2. **내 단계 캐릭터** — 현재 성장 단계(알/애벌레/번데기/벌/여왕벌) 큰 이미지
3. **진행 바** — 다음 단계까지 N개 남음
4. **우리 반 꿀벌집** — 반 전체 스티커 합계, 반 친구들 랭킹(상위 3명)
5. **최근 받은 칭찬** — 타임라인 (언제, 어떤 스티커, 선생님 메시지)

### 선생님이 보는 것
1. **학생 리스트 + 스티커 지급 버튼** — 이름별 카드, +1 / 종류 선택
2. **반 전체 누적 현황** — 학생별 스티커 수 막대 그래프
3. **스티커 취소** (실수 시 되돌리기, 5분 내)

## 2. 성장 단계 (Level System) — 확정

| 단계 | 한국어 | 이미지 | 누적 범위 | 시각 변화 |
|-----|--------|-------|---------|----------|
| 1 | **알** (egg) | 하얀 알 | 0 ~ 2개 | 꿀벌집 텅 빔 |
| 2 | **애벌레** (larva) | 통통한 노란 애벌레 | 3 ~ 5개 | 1칸씩 채워짐 |
| 3 | **번데기** (pupa) | 황금 번데기 | 6 ~ 15개 | 셀이 황금 테두리 |
| 4 | **벌** (bee) | 날개 달린 꿀벌 | 16 ~ 29개 | 꿀이 흘러넘침 |
| 5 | **여왕벌** (queen) | 왕관 쓴 꿀벌 | 30개+ | 육각 후광 + 별 |

**레벨 안에서도 진행 바**: 예) 벌 단계 22개 → "다음: 여왕벌 (8개 남음)"

**이유**: 생물학적 실제 곤충 성장 순서 (알→애벌레→번데기→성충). 교육적 가치 포함. 여왕벌은 리더십 상징.

## 2.1 레벨업 리워드 — 확정 (아이템 해금)

여왕벌까지 오르는 과정이 지루하지 않게, **단계별 해금**으로 커스터마이즈 가능.

### 해금 구조
```
애벌레 달성 → 색상 스킨 2종 해금 (주황, 연두)
번데기 달성 → 색상 스킨 4종 더 + 🎩 모자 악세서리
벌 달성   → 🧢/🎀/👑 악세서리 + 🐶 펫 1종 (강아지)
여왕벌 달성 → 🐱 🐰 🦋 펫 + 🏆 트로피 악세서리 + 후광
```

### 커스터마이즈 슬롯 (학생 본인이 선택)
- **Skin** (몸 색): 기본 노랑 / 주황 / 연두 / 하늘 / 분홍 / 보라 (6종, 단계별 해금)
- **Hat** (모자): 🎩 / 🧢 / 🎀 / 👑 (4종)
- **Pet** (옆 친구): 🐶 / 🐱 / 🐰 / 🦋 (4종)
- **Trophy** (트로피): 🏆 / ⭐ (여왕벌 전용)

### 데이터
```
rooms/{roomCode}/students/{clientId}/
  ├─ cosmetics: { skin, hat, pet, trophy }   // 현재 장착
  └─ unlocked:  { skins:[...], hats:[...], pets:[...], trophies:[...] }
```

장착은 학생 본인만 가능. 해금은 `stickerCount` 기준으로 자동 (계산형, 저장 안 함).

**UI**: 꿀벌집 화면에서 현재 단계 캐릭터 옆 "✨ 꾸미기" 버튼 → 장착 시트 팝업.

## 3. 데이터 모델 (Firebase)

### 경로 구조
```
rooms/{roomCode}/stickers/
  ├─ byStudent/
  │    └─ {clientId}/
  │         └─ {stickerId}: {
  │              id, type, fromTeacherName, fromTeacherId,
  │              message?, timestamp
  │            }
  └─ meta/
       └─ totalCount, lastGivenAt
```

### 스티커 타입 (초기)
- `helpful` 💛 "친절해요"
- `brave` 🧡 "용감해요"
- `creative` 🌸 "반짝이는 아이디어"
- `cooperative` 🤝 "함께 잘해요"
- `persistent` 🌱 "포기하지 않아요"
- `curious` 🔍 "궁금해해요"

6종 시작, 나중에 확장 가능.

### 권한
- **읽기**: 교실의 모든 구성원 (본인 것 + 반 통계)
- **쓰기**: 교사만 (`isTeacher` 검증)
- **삭제**: 교사만 + 5분 이내 (되돌리기)
- Firebase rules는 클라이언트 믿는 구조 유지 (기존 패턴과 동일, teacherCode 검증)

## 4. UI 구조

### 위치 선택
**→ 허브의 "우리반 현황판" 자리를 "칭찬 꿀벌집"으로 리네이밍**

허브 4번 섹션을 "📊 우리반 현황판" → "🍯 칭찬 꿀벌집" 으로 바꾸고, 내부는 탭 3개 (선생님은 4번째):
- 🍯 **나의 꿀벌집** (기본, 개인 캐릭터 + 진행 바 + 꿀벌집 + 꾸미기)
- 🏆 **개인전** (학생 개별 순위 랭킹 — 스티커 수 높은 순)
- 🏫 **단체전** (반 전체가 한 꿀벌집을 같이 채우는 별도 시스템 — 아래 참조)
- 👩‍🏫 **관리** (선생님 전용: 스티커 주기, 시즌 리셋, 학생별 통계)

### 개인전 vs 단체전 (확정 — 별도 시스템)

#### 개인전 (Individual Race)
- 각 학생이 **자신만의 꿀벌집** 을 채움
- 성장 단계(알→여왕벌) 있음
- 개인별 상위 랭킹 공개 (1위/2위/3위 아바타 + 이름)
- **커스터마이즈 해금은 개인전 스티커 기준**

#### 단체전 (Class Quest)
- 반 전체가 **하나의 큰 꿀벌집** 을 공동으로 채움
- 선생님이 **목표치 설정** (예: 100개 달성 시 반 파티)
- 진행 바 = 반 전체 누적 / 목표
- 달성 시 **반 전체 축하 이벤트** (화면 색종이 폭발 + "우리반 해냈다!")
- 개인 순위 **없음** — 같이 해내는 협동 메커니즘

### 데이터 분리

```
rooms/{roomCode}/stickers/
  ├─ individual/
  │    └─ {clientId}/{stickerId}: {...}
  ├─ team/
  │    └─ {stickerId}: {from, type, message?, ts}  (익명 누적만)
  └─ goal/
       ├─ target: 100
       └─ seasonStart: {timestamp}
```

스티커 줄 때 선생님이 **"개인전으로 줄까 / 단체전에 기여"** 를 먼저 선택.

### 화면별 레이아웃

#### 나의 꿀벌집 (학생 뷰)
```
┌─────────────────────────────┐
│  ← 뒤로        🍯 내 꿀벌집  │
├─────────────────────────────┤
│                             │
│     [대형 단계 이미지]        │  ← 현재 단계 (알/애벌레/…)
│     애벌레 🐛                │
│     12개 모았어요!            │
│                             │
│  [───── 진행바 ─────]        │  다음: 번데기까지 3개
│                             │
├─────────────────────────────┤
│  🍯 꿀벌집                   │
│  ⬡ ⬡ ⬡ ⬡ ⬡ ⬡              │  ← 육각 셀, 채워진 칸엔 꿀벌
│  ⬡ 🐝 🐝 🐝 ⬡ ⬡              │
│  🐝 🐝 🐝 🐝 🐝 🐝            │
│  🐝 🐝 🐝 🐝 🐝 🐝            │
│                             │
├─────────────────────────────┤
│  📜 최근 받은 칭찬            │
│  💛 친절해요 — 선생님 오전 10:34│
│  🌱 포기하지 않아요 — 어제    │
└─────────────────────────────┘
```

#### 우리반 꿀벌집 (모두 뷰)
- **대형 반 꿀벌집** (학생 전체의 스티커 합)
- **학생 리더보드**: 상위 5명 아바타 + 개수, 나머지는 수평 막대
- **오늘 받은 스티커 수** 빅 숫자

#### 스티커 주기 (선생님 뷰)
- 학생 이름별 카드 (아바타 + 현재 개수)
- 각 카드 클릭 → 모달:
  - 스티커 종류 6개 버튼
  - 선택 → "메시지 (선택)" 입력
  - 🐝 지급
- 지급 시 **색종이 + 꿀벌 날아가는 애니메이션** + 해당 학생에게 푸시 알림(선택)

## 8. 주의 & 리스크

### 공평성 이슈
- **문제**: 일부 학생만 많이 받음 → 소외감
- **대응**: 선생님 대시보드에 **"스티커 못 받은 학생" 하이라이트** 표시
- **대응 2**: 학생 본인에게는 **본인 진행 바만** 강조 (타인과 직접 비교 줄이기)

### 과잉 게이미피케이션
- **문제**: 스티커를 위해 행동하는 외재적 동기만 남음
- **대응**: 스티커 메시지 필수 입력 유도 → "왜" 가 남아있게 함
- **대응 2**: 여왕벌 도달해도 다시 시작 안 함 (은퇴 없음, 평생 유지)

### 데이터 프라이버시
- **문제**: 반 전체 뷰에서 다른 학생 기록 노출
- **대응**: 학생 이름은 이니셜/별명 옵션, 완전 익명 뷰 토글

- **2단계 확인** (한 번 실수 방지): "정말 모든 스티커를 초기화할까요?" → "네, 리셋해요"

- 한 학생에게 여러 개 몰아주기 가능

### 추가: 스티커 취소
- 각 학생 카드 확장 시 **최근 스티커 목록** 표시
- 5분 이내 지급 스티커 옆 🗑 버튼
